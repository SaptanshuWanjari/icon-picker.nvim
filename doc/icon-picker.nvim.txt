*icon-picker.nvim*  Icon picker for React projects

INTRODUCTION ~
  icon-picker.nvim provides a Telescope or snacks.nvim picker for icons from:
  - lucide-react
  - react-icons
  - @iconify-json/* with @iconify/react

FEATURES ~
  - Inserts JSX icon at cursor
  - Adds import safely
  - Handles multiline import blocks
  - Optional terminal/image preview

SETUP ~
>
  require("icon_picker").setup({
    command = "IconPicker",
    debug_command = "IconPickerDebug",
    keymaps = {
      normal = "<leader>ii",
      insert = "<C-g>i",
    },
    picker = {
      ui = "telescope",  -- or "snacks"
      preview_cache_max_age = 60 * 60 * 24 * 7,
    },
  })
<

COMMANDS ~
  :IconPicker
      Open picker.

  :IconPickerDebug [icon]
      Open a scratch debug report for preview issues. Optional icon tests a
      specific icon render path.

MAPPINGS ~
  Configure through setup(). No default keymaps unless provided.

IN-PICKER ~
  <C-t>  Reopen source selector when multiple sources are available.

PREVIEW ~
  Telescope previews use chafa in a terminal previewer.
  snacks.nvim previews use snacks.image when available. Enable:
>
  require("snacks").setup({
    picker = { enabled = true },
    image = { enabled = true },
  })
<
  snacks image previews also need a supported terminal and ImageMagick.
  Rendered SVG/PNG preview files are cached in:
>
  stdpath("cache")/icon-picker.nvim
<
  Cache cleanup runs once per Neovim session. Set preview_cache_max_age = 0 to
  disable cleanup.

DEBUGGING ~
  Run:
>
  :IconPickerDebug
<
  Or test a specific icon:
>
  :IconPickerDebug FcGoogle
<
  The report includes source counts, executable checks, picker backend checks,
  snacks image terminal support, render exit status, and cache file paths.
  For snacks image issues, also run:
>
  :checkhealth snacks
<

ICONIFY ~
  Iconify support reads installed @iconify-json/{prefix}/icons.json packages
  from node_modules. Selecting an Iconify icon inserts:
>
  <Icon icon="prefix:name" />
<
  and adds:
>
  import { Icon } from "@iconify/react";
<
